P2P Secure Print — Shop

This page receives ciphertext directly from the user via a WebRTC DataChannel. Keys are derived via ECDH and decryption happens only in memory.

1) WebRTC Signaling

If the User created an Offer, paste it below and click Set Offer. Then copy the Answer SDP and give it back to the User.

Connection status: idle

2) Decrypt & Print

When a file is received, this page will attempt to unwrap the file key, decrypt the bytes in memory, verify the SHA-256 fingerprint, then open a sandboxed iframe for printing (no download, no preview allowed outside sandbox).
